# Key Decisions

## Почему PixiJS вместо React
- React борется с D3/PixiJS за управление DOM/Canvas
- Каждый WS-тик → setState → re-render дерева → тормоза
- PixiJS даёт прямой контроль над WebGL/Canvas, zero overhead
- TradingView, Binance, wintrading — все используют Canvas/WebGL, не React

## Почему PixiJS вместо D3.js (SVG)
- SVG тормозит при >500 свечей на экране
- WebGL (PixiJS) — аппаратное ускорение, тысячи свечей без лагов
- Анимации, перетаскивание, зум — встроенные возможности PixiJS

## Почему PixiJS вместо raw Canvas2D
- Сцена (Scene Graph) — хиты, клики по свечам из коробки
- Текстуры, спрайты, анимации — встроенные
- Canvas2D пришлось бы писать весь хит-тест руками

## Почему TypeScript вместо JS
- Автокомплит в IDE
- Ошибки типов до рантайма (вместо `undefined is not a function`)
- Рефакторинг безопаснее

## Почему Vite вместо самописного
- HMR (изменения в коде → сразу в браузере)
- Сборка с минификацией
- Прокси на Python-бэкенд из коробки

## Почему tmux вместо systemd
- Проще, не требует root
- Один проект — одна tmux-сессия
- systemd — для продакшена, когда появится

## Почему V2 в отдельной папке, а не перезапись V1
- V1 работает (порт 5000), V2 — постепенная замена
- Можно быстро откатиться
- Сравнивать поведение параллельно
